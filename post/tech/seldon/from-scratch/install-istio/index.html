  2. Helm

1. Minikube

사용한 minikube version은 다음과 같습니다.

1> minikube version
2minikube version: v1.20.0
3commit: c61663e942ec43b20e8e70839dcca52e44cd85ae

minikube의 default config는 memory의 경우 2048mb 입니다. 이 경우 실습 중 메모리가 부족해서 OOM이슈가 생겨서 정삭적으로 진행할 수 없을 수 도 있습니다. OOM을 방지하기 위해서 minikube의 메모리를 4096mb로 늘려줍니다.

1> minikube config set memory 4096
2❗  These changes will take effect upon a minikube delete and then a minikube start

minikube를 실행시킵니다.

2. Helm

helm 공식 홈페이지의 방법을 따라 합니다. 이 때 버전은 3 이상이어야 합니다.
여러 방법중 스크립트 방법을 이용해 설치하겠습니다.

1curl -fsSL -o get_helm.sh <https://raw.githubusercontent.com/helm/helm/master/scripts/get-helm-3>
2chmod 700 get_helm.sh
3./get_helm.sh

3. Ingress

minikube에서 ingress addon 을 사용합니다.

1> minikube addons enable ingress

3   ▪ Using image docker.io/jettech/kube-webhook-certgen:v1.5.1
4   ▪ Using image docker.io/jettech/kube-webhook-certgen:v1.5.1

6🌟  The 'ingress' addon is enabled

4. Istio

Istio 공식홈페이지 가이드를 따라합니다.

Download Istio

스크립트를 이용해 다운로드합니다.

1curl -L <https://istio.io/downloadIstio> | sh -
2cd istio-1.9.5
3export PATH=$PWD/bin:$PATH

Install Istio

istioctl을 이용해 설치합니다.

1> istioctl install --set profile=demo -y
2✔ Istio core installed
3✔ Istiod installed
4✔ Egress gateways installed
5✔ Ingress gateways installed
6✔ Installation complete

default namespace에 istio-injection을 시켜서 istio-system이 생성되게 합니다.

1> kubectl label namespace default istio-injection=enabled
2namespace/default labeled

ingress ip and ports

  • ingress port 설정하기

    1export INGRESS_PORT=$(kubectl -n istio-system get service istio-ingressgateway -o jsonpath='{.spec.ports[?(@.name=="http2")].nodePort}')
2export SECURE_INGRESS_PORT=$(kubectl -n istio-system get service istio-ingressgateway -o jsonpath='{.spec.ports[?(@.name=="https")].nodePort}')

  • port 확인하기

    1> echo "$INGRESS_PORT"
230687

    1> echo "$SECURE_INGRESS_PORT"
230520

  • Ingress ip 설정하기

    1export INGRESS_HOST=$(minikube ip)

  • ip 설정 확인하기

    1> echo "$INGRESS_HOST"
2192.168.49.2

Seldon-core

Install seldon-core

  • namespace 생성

    1> kubectl create namespace seldon-system
2
3namespace/seldon-system created

  • helm chart 생성

    1helm install seldon-core seldon-core-operator \\
2    --repo <https://storage.googleapis.com/seldon-charts> \\
3    --set usageMetrics.enabled=true \\
4    --namespace seldon-system \\
5    --set istio.enabled=true

  • 상태 확인

    1> kubectl get po -n seldon-system
2NAME                                        READY   STATUS    RESTARTS   AGE
3seldon-controller-manager-559c567c9-pjtpl   1/1     Running   0          11s

Istio setting

  • istio-ingress.yaml

     1## istio-ingress.yaml
 2apiVersion: networking.istio.io/v1alpha3
 3kind: Gateway
 4metadata:
 5  name: seldon-gateway
 6  namespace: istio-system
 7spec:
 8  selector:
 9    istio: ingressgateway ## use istio default controller
10  servers:
11  - port:
12      number: 80
13      name: http
14      protocol: HTTP
15    hosts:
16    - "*"

  • 다음과 같은 istio ingress 를 생성 후 apply 합니다.

    1> k apply -f istio-ingress.yaml
2gateway.networking.istio.io/seldon-gateway created

Sample deploy

  • namespace 생성

    1kubectl create namespace seldon

  • pod 생성

     1kubectl apply -f - << END
 2apiVersion: machinelearning.seldon.io/v1
 3kind: SeldonDeployment
 4metadata:
 5  name: iris-model
 6  namespace: seldon
 7spec:
 8  name: iris
 9  predictors:
10  - graph:
11      implementation: SKLEARN_SERVER
12      modelUri: gs://seldon-models/sklearn/iris
13      name: classifier
14    name: default
15    replicas: 1
16END

  • minikube tunnel

     1> minikube tunnel
 2Status:
 3        machine: minikube
 4        pid: 117019
 5        route: 10.96.0.0/12 -> 192.168.49.2
 6        minikube: Running
 7        services: [istio-ingressgateway]
 8    errors: 
 9                minikube: no errors
10                router: no errors
11                loadbalancer emulator: no errors

  • ingress external ip 확인하기

    1> k get svc -n istio-system
2NAME                   TYPE           CLUSTER-IP      EXTERNAL-IP     PORT(S)                                                                      AGE
3istio-egressgateway    ClusterIP      10.104.239.89   <none>          80/TCP,443/TCP,15443/TCP                                                     7m43s
4istio-ingressgateway   LoadBalancer   10.111.24.207   10.111.24.207   15021:30945/TCP,80:30687/TCP,443:30520/TCP,31400:30072/TCP,15443:32717/TCP   7m43s
5istiod                 ClusterIP      10.99.171.243   <none>          15010/TCP,15012/TCP,443/TCP,15014/TCP                                        7m58s

    → 이 경우 10.111.24.207

  • predict 요청 날려보기

    1curl -X POST <http://10.111.24.207/seldon/seldon/iris-model/api/v1.0/predictions> \\
2    -H 'Content-Type: application/json' \\
3    -d '{ "data": { "ndarray": [[1,2,3,4]] } }'
    • format
      • http://<ingress_url>/seldon/<namespace>/<model-name>/api/v1.0/doc/

  • 다음과 같은 결과를 얻을 수 있습니다.

    1{"data":{"names":["t:0","t:1","t:2"],"ndarray":[[0.0006985194531162841,0.003668039039435755,0.9956334415074478]]},"meta":{"requestPath":{"classifier":"seldonio/sklearnserver:1.7.0"}}}

Seldon-analytics

Install

  • helm chart 설치

    1helm install seldon-core-analytics seldon-core-analytics \\
2   --repo <https://storage.googleapis.com/seldon-charts> \\
3   --namespace seldon-system

Usage

  • 설치된 seldon-core-analytics port-forward

    1kubectl port-forward svc/seldon-core-analytics-grafana 3000:80 -n seldon-system

  • http://localhost:3000 에 접속합니다.

    • admin / password 로 로그인 합니다.
    • dashboard에서 prediction analytics를 클릭합니다
    • 다음과 같은 dashboard를 볼 수 있습니다.
