이번 포스트에서는 argocd를 이용해 github branch 에서 cd(continuous delivery) 하는 법에 대해서 알아보겠습니다.

1. github action 설정하기

1.1 CI

CI를 위한 github action을 작성하겠습니다.

 1  docker-image-ci:
 2    runs-on: ubuntu-18.04
 3    steps:
 4      - name: Checkout
 5        uses: actions/checkout@v2
 6        with:
 7          submodules: recursive
 8          fetch-depth: 0
 9      - name: Create shot SHA
10        uses: benjlevesque/short-sha@v1.2
11        id: short-sha
12      - name: Set up Docker Buildx
13        uses: docker/setup-buildx-action@v1
14      - name: Login to GitHub Container Registry
15        uses: docker/login-action@v1
16        with:
17          registry: ghcr.io
18          username: ${{ github.repository_owner }}
19          password: ${{ secrets.CR_PAT }}
20      - name: Setup python
21        uses: actions/setup-python@v2
22        with:
23          python-version: '3.8.5'
24      - name: Generate html
25        run: |
26          pip install -r requirements-doc.txt
27          cd docs
28          make html          
29      - name: Build and push
30        uses: docker/build-push-action@v2
31        with:
32          push: true
33          context: ./
34          file: docker/Dockerfile
35          tags: |
36            ghcr.io/aiden-jeon/sphinx-api:latest
37            ghcr.io/aiden-jeon/sphinx-api:${{ steps.short-sha.outputs.sha }}

main branch에 수정이 생기면 docs를 build하고 docker image를 만듭니다. docs를 build하기 위해서 requirements-doc.txt 를 추가합니다.

1# requirements-doc.txt
2sphinx-autoapi==1.6.0
3sphinx-rtd-theme==0.5.1

1.2 CD

CI를 위한 github action을 작성하겠습니다.

 1  docker-image-cd:
 2    runs-on: ubuntu-18.04
 3    steps:
 4      - name: Create short SHA
 5        uses: benjlevesque/short-sha@v1.2
 6        id: short-sha
 7      - name: Checkout deploy repo
 8        uses: actions/checkout@v2
 9        with:
10          repository: aiden-jeon/github-cicd
11          token: ${{ secrets.CR_PAT }}
12          path: github-cicd
13      - name: Install yq
14        run: |
15          wget https://github.com/mikefarah/yq/releases/download/v4.4.1/yq_linux_amd64 -O ./yq
16          chmod +x ./yq          
17      - name: Change image tag on values.yaml
18        run: |
19          cat ./github-cicd/sphinx-doc/values.yaml | ./yq e '.image.tag="${{ steps.short-sha.outputs.sha }}"' - | tee ./github-cicd/sphinx-doc/values.yaml.tmp
20          mv ./github-cicd/sphinx-doc/values.yaml.tmp ./github-cicd/sphinx-doc/values.yaml          
21      - name: Commit file
22        run: |
23          cd github-cicd
24          git config --local user.email "ells2124@gmail.com"
25          git config --local user.name "aiden-jeon"
26          git add ./sphinx-doc/values.yaml
27          git commit -m "Update sphinx-doc image tag."          
28      - name: Push changes
29        uses: ad-m/github-push-action@master
30        with:
31          directory: github-cicd
32          repository: aiden-jeon/github-cicd
33          github_token: ${{ secrets.CR_PAT }}
34          branch: doc-pages
35          force: true

CI가 종료되면 doc-pages branch 의 values를 수정합니다.
