이번 포스트에서는 helm 을 이용해 이전 포스트 에서 작성한 파일들을 자동화 하는 법에 대해서 알아 보겠습니다.

1. helm start

helm 을 시작하려는 repo 최상단에서 다음과 같이 입력해줍니다.

1> helm create sphinx-doc
2
3Creating sphinx-doc

그러면 아래와 같은 파일들이 생성됩니다.

1> cd sphinx-doc
2> ls
3
4Chart.yaml  charts      templates   values.yaml

3. service.yaml

생성된 service.yaml 을 다음과 같이 수정합니다.

 1{% raw %}
 2apiVersion: v1
 3kind: Service
 4metadata:
 5  namespace: sphinx-doc
 6  name: {{ template "sphinx-doc.fullname" . }}
 7  labels:
 8    app: {{ template "sphinx-doc.name" . }}
 9    chart: {{ template "sphinx-doc.chart" . }}
10    release: {{ .Release.Name }}
11    heritage: {{ .Release.Service }}
12spec:
13  type: {{ .Values.service.type }}
14  ports:
15    - port: {{ .Values.service.port }}
16      targetPort: http
17      protocol: TCP
18      name: http
19  selector:
20    app: {{ template "sphinx-doc.name" . }}
21    release: {{ .Release.Name }}
22{% endraw %}

4. ingress.yaml

생성된 ingress.yaml 을 다음과 같이 수정합니다.

 1{% raw %}
 2{{- if .Values.ingress.enabled -}}
 3{{- $fullName := include "sphinx-doc.fullname" . -}}
 4{{- $ingressPath := .Values.ingress.path -}}
 5{{- $svcPort := .Values.service.port -}}
 6{{- if semverCompare ">=1.14-0" .Capabilities.KubeVersion.GitVersion -}}
 7apiVersion: networking.k8s.io/v1beta1
 8{{- else -}}
 9apiVersion: extensions/v1beta1
10{{- end }}
11kind: Ingress
12metadata:
13  namespace: mrxflow
14  name: {{ template "sphinx-doc.fullname" . }}
15  labels:
16    app: {{ template "sphinx-doc.name" . }}
17    chart: {{ template "sphinx-doc.chart" . }}
18    release: {{ .Release.Name }}
19    heritage: {{ .Release.Service }}
20  {{- with .Values.ingress.annotations }}
21  annotations:
22    {{- toYaml . | nindent 4 }}
23  {{- end }}
24spec:
25{{- if .Values.ingress.tls }}
26  tls:
27  {{- range .Values.ingress.tls }}
28    - hosts:
29      {{- range .hosts }}
30        - {{ . | quote }}
31      {{- end }}
32      secretName: {{ .secretName }}
33  {{- end }}
34{{- end }}
35  rules:
36  {{- range .Values.ingress.hosts }}
37    - host: {{ .host }}
38      http:
39        paths:
40          - path: {{ $ingressPath }}
41            backend:
42              serviceName: {{ $fullName }}
43              servicePort: {{ $svcPort }}
44  {{- end }}
45{{- end }}

 1replicaCount: 1
 2image:
 3  repository: ghcr.io/aiden-jeon/sphinx-api
 4  tag: dc3c4be
 5  pullPolicy: Always
 6  pullSecrets: test-ghcr

 8  type: NodePort
 9  port: 80
10ingress:
11  enabled: false
12  annotations:
13    kubernetes.io/ingress.class: nginx
14    ingress.kubernetes.io/rewrite-target: /
15  path: /
16  hosts:
17    - host: chart-example.local
18  tls: []
19
20resources: {}

3. helm install

이제 helm 을 이용해 install 해보겠습니다.

우선 kubectl namespace를 생성합니다.

1> kubectl create ns sphinx-doc
2
3namespace/sphinx-doc created

새 namespace에 secret을 추가해줍니다.

1kubectl -n sphinx-doc create secret docker-registry test-ghcr \
2    --docker-server=ghcr.io \
3    --docker-username=aiden-jeon \
4    --docker-password=<secret_key> \
5    --docker-email=ells2124@gmail.com

사용하지 않는 serviceaccount.yaml hpa.yaml 을 삭제합니다.

1rm sphinx-doc/templates/serviceaccount.yaml
2rm sphinx-doc/templates/hpa.yaml

그리고 helm install을 합니다.

 1> helm install sphinx-doc sphinx-doc/ --values sphinx-doc/values-local.yaml
 2

 4LAST DEPLOYED: Thu Mar  4 16:59:41 2021
